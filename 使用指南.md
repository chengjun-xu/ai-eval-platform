# AI 大模型评测平台 — 使用指南

> ByteBrain Eval Platform：一个功能完备的大模型评测平台，支持多维度、多场景的模型评估。

---

## 一、平台概述

平台采用 Flask + Jinja2 架构，核心评测流程：

**选择模型 + Benchmark → 后台线程执行 → 调用 LLM API → 评分（规则/Judge/Rubric）→ Bootstrap 置信区间 → Bad Case 归因 → 数据污染检测 → 报告生成**

### 启动方式

```bash
git clone https://github.com/chengjun-xu/ai-eval-platform.git
cd ai-eval-platform
pip install flask requests
python3 app.py
# 浏览器访问 http://localhost:5001
```

首次使用需注册账号。每个用户的数据完全隔离。

## 二、快速上手（5 分钟）

### 步骤 1：注册模型

进入「**模型管理**」页面，填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| 模型名称 | 显示名称 | DeepSeek V4 Flash |
| 模型 ID | 唯一标识 | deepseek-v4-flash |
| 提供商 | 厂商名 | DeepSeek |
| API Base URL | 接口地址 | https://api.deepseek.com/v1 |
| API Key | 密钥 | sk-... |
| 模型名（API） | 传给 API 的参数 | deepseek-v4-flash |
| 描述 | 备注 | 快速版 |

> ⚠️ API Key 仅存储在本地 `data/models.json` 中，不会上传到 GitHub。该文件已被 `.gitignore` 排除。

### 步骤 2：注册 Judge 模型（可选）

进入「**Judge 模型**」页面，注册用于开放题评分的模型。建议使用比待评测模型更强的模型。

### 步骤 3：运行评测

进入「**运行评测**」页面：
1. 选择待评测模型
2. 选择 Judge 模型（OpenEval 时必选）
3. 勾选一个或多个 Benchmark
4. 选择模式：**完整评测**（全部题目）或 **快速回归**（分层抽样 20 题）
5. 点击「开始评测」

### 步骤 4：查看结果

评测报告包含：

- 综合得分 + 95% 置信区间
- 能力雷达图
- 各 Benchmark 得分柱状图
- 数据泄露风险评估
- Bad Case 归因分析
- 安全合规评分

## 四、评分机制

### 三种评分方式

| 方式 | 适用场景 | 机制 |
|------|----------|------|
| **规则评分** | 选择题/数学/编程 | 精确匹配、数字比较、代码执行 |
| **LLM-as-Judge** | 开放题 | Judge 模型 1-5 分 |
| **Rubric** | 复杂场景 | 多维度加权评分 |

### LLM-as-Judge 评分标准

| 分数 | 含义 |
|------|------|
| 5 分 | 优秀：准确、完整、结构清晰、见解深刻 |
| 4 分 | 良好：准确、完整、清晰 |
| 3 分 | 及格：基本正确但不够完善 |
| 2 分 | 较差：部分相关但信息不足 |
| 1 分 | 差：完全不相关或错误 |

### Rubric 预定义模板

| 模板 | 维度 |
|------|------|
| 通用 | 准确性、完整性、清晰度、相关性 |
| 医疗 🏥 | 医学准确性、安全性、完整性、可解释性 |
| 代码 💻 | 正确性、效率、可读性、健壮性 |
| 翻译 🌐 | 准确性、流畅度、术语一致性、风格 |
| 安全 🛡️ | 危害识别、拒绝恰当、立场正确、解释清晰 |

### 置信区间

使用 **Bootstrap 重采样（1000 次）** 计算 95% 置信区间。区间越窄，结果越稳定。

**快速回归模式**：分层抽样 20 题，适合日常迭代。

## 五、高级功能

### 数据泄露检测

n-gram 重叠分析（3/5/9/13-gram），风险等级：

安全 → 低 → 中 → 高 → 严重

### Bad Case 归因

自动将错误归因到 7 类失败模式，每类附改进建议：

| 失败模式 | 改进建议 |
|----------|---------|
| 📋 指令遵循失败 | SFT 数据增强 |
| 📚 知识缺失/遗忘 | 领域 SFT + RAG |
| 🔗 逻辑断裂 | CoT 训练数据 |
| 👻 幻觉模式 | DPO + 奖励模型 |
| 🚫 拒答策略不当 | 校准安全对齐边界 |
| 🔍 证据不一致 | RAG 约束数据 |
| 🔧 工具调用失败 | Agent 训练数据 |

### 模型对比

「**对比**」页面选择两个模型，系统自动生成双轴雷达图。

### 评测 Agent

「**Agent**」页面支持自然语言指令：

- "评测我的模型，跑全部"
- "帮我跑 MMLU 和 GSM8K"
- "测代码能力，用 Judge 评分"

### 自定义数据集

「**数据集**」页面上传 JSON，支持三种格式：

**选择题（MMLU 格式）：**
```json
{
  "id": "q1",
  "category": "医学",
  "question": "...",
  "choices": {"A": "...", "B": "...", "C": "...", "D": "..."},
  "answer": "A"
}
```

**数学题（GSM8K 格式）：**
```json
{
  "id": "q1",
  "question": "...",
  "answer": "42"
}
```

**编程题（HumanEval 格式）：**
```json
{
  "id": "q1",
  "prompt": "def foo():",
  "test": "assert foo() == 42",
  "description": "..."
}
```

## 七、常见问题

**Q：评测跑得很慢怎么办？**
A：使用「快速回归」模式（20 题）。完整评测建议非工作时间运行。

**Q：Judge 模型怎么选？**
A：使用比待评测模型更强的模型。若 Judge 不够强，评分可能不准。

**Q：评测结果不准确？**
A：检查数据泄露检测报告，增加样本量以获得更稳定的置信区间。

**Q：怎么添加自定义数据？**
A：在「数据集」页面上传 JSON 文件，支持三种格式。

**Q：如何理解置信区间？**
A：95% CI 表示重复评测 100 次，95 次得分落在此范围内。区间越窄越稳定。
